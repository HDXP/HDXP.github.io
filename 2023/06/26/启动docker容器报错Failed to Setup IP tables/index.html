启动docker容器报错Failed to Setup IP tables

发表于2023-06-26|更新于2023-06-27|docker
|阅读量:

背景

如果在docker服务启动之后，再执行关闭系统防火墙的操作，那后续执行启动docker容器（docker-compose up -d）或者创建docker 网络（docker create network），会出现下面的报错。

原因

Docker 在启动时会依赖 Linux的防火墙规则进行网络连接和流量转发。它会在 Linux 的 iptables 中创建一些规则，以便容器可以与主机和其他容器进行通信。这些规则是根据 Docker的网络配置动态生成的。

如果在 Docker 启动后关闭了 Linux 的防火墙，而 Docker在启动时已经创建了一些防火墙规则，此时关闭防火墙可能导致这些规则不再生效。这可能会导致容器之间无法互相访问或与主机进行通信。

为了确保 Docker 的网络连接正常，当关闭 Linux 的防火墙或修改防火墙规则后，建议重启 Docker 服务。通过重启Docker，它将重新创建所需的防火墙规则，并根据当前的网络配置进行更新。
